# SKILL_POLICY.md - Skills 谁制定、怎么用

## 谁提出 skill

Worker 在具体工作中发现重复模式、坑、稳定流程时，提出 skill_update_request。

## 谁整理 skill

中书收集多个 worker 的建议，交给 harness_worker 整理成 skill 草案。

## 谁批准 skill

领袖判断是否进入长期框架。

## Skill 作用

- 让低价/基础 worker 获得稳定工作方法。
- 固化工程经验。
- 降低中书反复解释成本。
- 避免相同 bug 和流程错误重复出现。

## Skill 必须包含

- 触发条件
- 输入要求
- 操作步骤
- 输出格式
- 常见坑
- 验证方式
- 适用 worker
