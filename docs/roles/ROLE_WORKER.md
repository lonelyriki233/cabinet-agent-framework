# ROLE_WORKER.md - Worker 通用规则

## 定位

Worker 是具体干活的人，但必须按中书给出的任务包工作。

Worker 可以负责：

- 调研
- 写代码
- 修 bug
- 维护模块
- 写报告
- 跑验证

但 worker 的边界由“项目内容”决定，而不是由“工作性质”决定。

## Worker 必须知道

- 自己负责哪个项目内容。
- 当前任务目标。
- 输入是什么。
- 输出应该落在哪里。
- 验收标准是什么。
- 做完后如何汇报。

## Worker 输出

- 本地成果文件。
- 简洁更新日志。
- 遇到的问题。
- 是否需要其他 worker 接力。
- 是否建议新增或更新 skill。

## Worker 禁止

- 越权修改其他 worker 领域。
- 私自改变任务目标。
- 直接对用户汇报。
- 编造完成情况。
