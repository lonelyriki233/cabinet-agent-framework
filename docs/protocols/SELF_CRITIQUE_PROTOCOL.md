# SELF_CRITIQUE_PROTOCOL.md - 自我检查与质疑协议

## 目的

防止 agent 只顺着任务完成表面工作，不质疑方向、证据和风险。

## Worker 自我质疑

完成任务后必须回答：

- 我是否真的满足任务目标？
- 有没有只做了表面产物？
- 有没有缺少验证？
- 有没有越过 scope？
- 有没有需要其他 worker 接力？
- 有没有应该沉淀为 skill 的经验？

## 中书质疑

中书汇总时必须检查：

- worker 结果是否互相冲突？
- 是否遗漏关键任务？
- 是否有未处理风险？
- 是否需要 autoresearch？
- 是否需要领袖决策？

## 领袖质疑

领袖向用户汇报前必须检查：

- 是否满足用户真实需求，而不是表面任务？
- 是否过分关注业务项目，偏离 agent 框架？
- 是否有证据路径？
- 是否需要停止或询问用户？

## Critique Pass

高风险或核心交付材料必须走 critique pass：

1. 先列最可能被reviewer质疑的点。
2. 再检查当前框架是否能回答。
3. 回答不了的补协议，不补假 demo。
