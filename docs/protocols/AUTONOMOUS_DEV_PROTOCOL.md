# AUTONOMOUS_DEV_PROTOCOL.md - 挂机/夜间开发协议

## 原则

挂机开发不是让 agent 无限发挥，而是按已授权任务队列小步推进。

## 启动前提

必须同时满足：

- 任务已由领袖授权。
- 中书已拆成任务包。
- 每个任务有 worker、输入、输出、验收标准、停止条件。
- 明确 forbidden scope。

## 每轮循环

1. 中书读取当前项目状态。
2. 选择一个 ready 任务。
3. 派给对应 worker。
4. worker 只推进一个明确阶段。
5. worker 写本地成果和更新日志。
6. 中书汇总并更新状态。
7. 运行必要 hooks/checks。
8. 失败、越权、不确定、高风险时停止。

## Stop Condition

任一满足即停止：

- 任务完成。
- 检查失败。
- 遇到 forbidden scope。
- 需要用户决策。
- 成本/时间超过预算。
- worker 连续两轮没有实质进展。

## 早晨汇报

中书给领袖：

- 推进了什么。
- 哪些 worker 工作过。
- 产物路径。
- 检查结果。
- blocker。
- 下一步建议。
