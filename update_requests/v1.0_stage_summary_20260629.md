# CAF v1.0 阶段性总结

时间：2026-06-29

## 当前阶段判断

CAF 已经从“材料/目录/协议较多，但强运行闭环不足”的状态，推进到 Agent Project Harness v1.0 的可运行 MVP 阶段。

现在 CAF 已经具备一条真实闭环：

```text
需求输入
→ Harness Project 创建
→ HarnessTask 创建
→ Worker Runtime 生成执行上下文
→ Artifact Registry 登记产物
→ Gate System 审查
→ Context / Memory Engine 召回上下文
→ Showcase 展示框架能力
```

## 已完成模块

### 1. Project Harness Core

实现位置：

```text
cabinet_framework/project_harness.py
```

能力：

- project
- task
- artifact
- gate event
- project decision
- runtime 状态统计
- demo loop

### 2. Worker Runtime

实现位置：

```text
cabinet_framework/worker_runtime.py
```

能力：

- 为 HarnessTask 生成 worker prompt
- 登记 prompt 为 logs artifact
- 接收 worker manifest
- 自动登记 worker 产物
- 支持 run + gate 联动

### 3. Harness Intake

实现位置：

```text
cabinet_framework/harness_intake.py
cabinet_framework/conversation_bridge.py
```

能力：

- 普通 request / chat-intake 可自动进入 Harness
- 创建 HarnessProject
- 创建 HarnessTask
- 保留 legacy task packet 兼容

### 4. Context / Memory Engine

实现位置：

```text
cabinet_framework/context_engine.py
```

能力：

- 生成 context pack
- 召回 memory object
- 注入 project decisions
- 注入 related artifacts
- 注入 gate history
- 注入 allowed context
- worker prompt 自动使用 context pack

### 5. Showcase 展示页

实现位置：

```text
showcase/
```

技术栈：

- Vite
- React
- TypeScript

- 不是 runtime dashboard

## 当前还缺什么

### 1. Gate / Hook / Lifecycle 合并

现在 gate 可运行，但仍偏独立。

需要变成任务生命周期硬约束：

```text
task stage transition
→ run gate
→ pass 才能进入下一阶段
→ fail 自动生成 blocker / rework task
```

### 2. Rework / Blocker 闭环

现在 gate fail 会返回 fail，但还没有自动生成返工任务。

需要补：

- blocker record
- rework task
- parent/child task relation
- retry history

### 3. Archive / Learning 闭环

现在已有 archive 和 memory object，但还没完全自动化。

需要补：

- gate fail 自动沉淀 memory case
- completed task 自动归档
- validated artifact 自动归档索引
- reusable lesson 可导出为 skill candidate

### 4. Worker 实执行器

现在 worker runtime 是 prompt/manifest 协议层，尚未直接调用真实 agent lane。

后续可接：

- Hermes delegate_task
- Codex lane
- Claude Code lane
- local command worker
- CI/headless worker

### 5. Tool / MCP Contract 加硬

当前已有 ToolContract 雏形，但未强绑定到 worker runtime。

需要补：

- worker allowed tools
- tool risk level
- dry-run / approval policy
- side-effect gate

## 下一步计划

优先顺序：

### Step 1：Gate / Hook / Lifecycle 合并

目标：让 gate 成为状态推进硬约束。

交付：

```text
cabinet_framework/lifecycle_gate.py
tests/test_lifecycle_gate_v1.py
docs/core/LIFECYCLE_GATE.md
update_requests/v1.0_update_*_lifecycle_gate.md
```

### Step 2：Rework / Blocker 闭环

目标：gate fail 后自动形成可执行返工任务。

交付：

```text
blocker registry
rework task generation
parent_task_id / retry_of metadata
```

### Step 3：Archive / Learning 自动化

目标：任务完成后自动沉淀 artifact、decision、失败经验。

交付：

```text
auto archive
memory case export
skill candidate export
```

### Step 4：真实 Agent Worker Lane

目标：让 HarnessTask 能调用真实 worker，而不只是生成 prompt。

交付：

```text
worker lane adapter
structured worker output
cost / timeout / scope policy
```

## 当前建议

展示页保持最低限度即可，不继续投入太多展示层工作；CAF v1.0 当前优先级应全部转向功能性闭环。

下一步先做：

```text
Gate / Hook / Lifecycle 合并
```

原因：

这是把 CAF 从“能跑流程”推进到“能约束流程”的关键一步。没有这一层，worker、memory、artifact 都能存在，但项目仍然可能绕过 gate 直接声称完成。
