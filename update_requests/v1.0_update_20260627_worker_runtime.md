# CAF v1.0 更新记录：Worker Runtime 接入

时间：2026-06-27

## 本次更新

接续 `v1.0_update_20260627_project_harness.md`，把 HarnessTask 接入 Worker Runtime。

现在链路变为：

```text
HarnessTask
→ Worker Prompt 生成
→ Prompt 登记为 logs artifact
→ Worker 提交 JSON manifest
→ Artifact Registry 登记领域产物
→ Gate System 审查
```

## 新增 CLI

```bash
python3 -m cabinet_framework.cli harness-run --task-id <T> --mode prompt
python3 -m cabinet_framework.cli harness-run --task-id <T> --mode local-report
python3 -m cabinet_framework.cli harness-submit --manifest manifest.json
python3 -m cabinet_framework.cli harness-run-gate --task-id <T> --mode local-report
```

## 设计边界

`local-report` 只验证 Worker Runtime 管道，不伪造领域产物。

如果任务要求 `docs/code/tests/design` 等领域产物，Gate 仍会失败，直到 worker 通过 manifest 提交真实 artifact。
