# CAF v1.0 更新记录：Context / Memory Engine 接入

时间：2026-06-29

## 本次更新

继续完善 CAF v1.0 本体，补上 Context / Memory Engine 的第一版可运行实现。

现在 worker 不再只拿 allowed_paths 摘要，而是拿到结构化 Context Pack。


## 新增能力

### 1. Context Pack

每个 HarnessTask 可以生成：

```text
runtime/project_harness/context_packs/<task_id>.json
```

内容包括：

```text
project
task
allowed_context
project_decisions
related_artifacts
gate_history
recalled_memory
context_rules
```

### 2. Worker Prompt 自动接入 Context Pack

`harness-run` / `prepare_harness_worker` 现在会自动生成 context pack，并把以下内容注入 worker prompt：

```text
project decisions
allowed context
related artifacts
gate history
recalled memory
context rules
```

### 3. Memory Object 轻量召回

新增轻量 keyword recall，用于 v1.0 MVP。

当前不是向量数据库，先保证可运行、可审计、可测试。

### 4. CLI

新增：

```bash
python3 -m cabinet_framework.cli harness-context --task-id <TASK>
python3 -m cabinet_framework.cli harness-context --project-id <PROJECT>
python3 -m cabinet_framework.cli harness-memory --task-id <TASK> --title "..." --content "..."
```

## 验收标准

- context pack 能生成
- worker prompt 包含 context pack
- memory object 能登记并召回
- 测试通过
- CAF gate 通过
- Showcase build 通过

## 遗留问题

1. 还不是向量召回。
2. 还没有 conflict detection。
3. gate failure 还没有自动沉淀成 memory case。
4. project-local memory namespace 还需要强化。
