# CAF v1.0 更新记录：Lifecycle Gate 接入

时间：2026-06-29

## 本次更新

继续完善 CAF 功能性闭环，不扩展示示页。

本次把 Gate System 接入 HarnessTask 生命周期，让 gate 从“独立检查命令”变成 stage 推进硬约束。

## 核心变化

以前：

```text
worker 产出 artifact
→ 可手动运行 gate
→ gate 结果只作为检查
```

现在：

```text
HarnessTask 当前 stage
→ harness-advance
→ 自动运行当前 stage gate
→ pass 才推进 stage
→ fail 阻止推进，记录 blocker，生成 rework task
```

## 通过路径

如果 gate pass：

```text
task.stage = target_stage
task.status = ready / completed
transition.status = advanced
hook = harness.lifecycle.transition_advanced
```

## 失败路径

如果 gate fail：

```text
task.stage 不变
task.status = blocked
blocker 写入 blockers.jsonl
rework task 自动创建
transition.status = blocked
hook = harness.lifecycle.transition_blocked
```

## 验证标准

1. gate fail 不允许推进 stage。
2. gate fail 自动记录 blocker。
3. gate fail 自动生成 rework task。
4. gate pass 后推进 stage。
5. CLI 能调用生命周期推进。
6. 全量测试通过。
7. CAF gate 通过。

## 对真实工程流程的意义

这个更新让 CAF 更契合真实工程维护流程：

- 软件开发：测试/review/delivery 不通过不能进入下一阶段。
- 工业 agent：安全/权限/动作 gate 不通过不能执行下一步。
- 门店管理 agent：日报、库存、排班建议必须经过业务规则 gate。
- 机器人/VLA：action proposal 必须经过 safety/simulation gate。

CAF 仍不做垂直业务逻辑，而是强化共通开发维护流程。
