# CAF v1.0 更新记录：Harness Intake 接入

时间：2026-06-28

## 本次更新

把普通需求接入 v1.0 Project Harness。

现在链路变为：

```text
chat/intake
→ LeaderDecision / Mandate
→ HarnessProject
→ HarnessTask(s)
→ Worker Runtime
→ Artifact Registry
→ Gate System
```

legacy TaskPacket 仍保留，用于兼容旧 runtime/dashboard。

## 新增文件

```text
cabinet_framework/harness_intake.py
docs/core/HARNESS_INTAKE.md
tests/test_harness_intake_v1.py
```

## 修改文件

```text
cabinet_framework/conversation_bridge.py
cabinet_framework/cli.py
cabinet_framework/project_harness.py
```

## 新增 CLI

```bash
python3 -m cabinet_framework.cli harness-intake --request "..." --authority L2
```

## 行为变化

`chat-intake` 现在会返回并登记：

```text
harness.project
harness.tasks
```

也就是说，普通对话需求不再只停留在 legacy TaskPacket，而是直接进入 v1.0 harness。

## 验证命令

```bash
cd <repo-root>
python3 -m unittest tests.test_harness_intake_v1 -v
python3 -m cabinet_framework.cli harness-intake --request "完善 CAF worker runtime 和 gate 闭环" --authority L2
python3 -m unittest discover -s tests -v
python3 -m cabinet_framework.cli gate
```

## 遗留问题

1. Dashboard 还没展示 harness project/task/artifact/gate。
2. harness-intake 的任务拆解仍复用 legacy secretariat，需要逐步升级为 Project Harness 原生拆解器。
3. Gate 还没完全合并到 hooks/lifecycle stop gate。
